## Backend Infrastructure

### Stack
- **Runtime**: TanStack Start server functions (already in place)
- **Database**: PostgreSQL via Drizzle ORM (`drizzle-orm/node-postgres`)
- **Auth**: Better Auth (self-hosted, fits VPS deployment)
- **Deployment**: Docker on VPS (Dockerfile already scaffolded)

### Database Schema (Postgres)

Mirror the capacitor app's SQLite schema. One account → many devices, each device has its own copy of the data. Sync merges by `updated_at` timestamp (last-write-wins).

```
users              — Better Auth managed
accounts           — Better Auth managed (OAuth providers if added later)
sessions           — Better Auth managed

sync_books         — mirrors capacitor `books` table, keyed by (user_id, book_id)
sync_settings      — mirrors capacitor `settings` table, keyed by user_id
sync_highlights    — mirrors capacitor `highlights` table, keyed by (user_id, highlight_id)
```

`sync_books` columns: `user_id`, `book_id` (client-generated UUID), `title`, `author`, `file_size`, `word_count`, `cover_url`, `position` (byte offset), `updated_at`

> Book *files* are **not** stored server-side (too large, privacy concern). Only metadata + position.

### API Endpoints (TanStack Start server functions)

**Auth** (Better Auth handles these automatically):
- `POST /api/auth/sign-up`
- `POST /api/auth/sign-in`
- `POST /api/auth/sign-out`

**Sync**:
- `GET  /api/sync` — pull full sync state for the logged-in user
- `POST /api/sync` — push local state (full snapshot), merge server-side by `updated_at`

> Simple full-snapshot sync first. Delta sync can come later if the payload gets large.

## Sync Architecture (App ↔ Server)

The capacitor app already has a complete local SQLite database. Sync = push/pull that state when the user is logged in.

### Flow
1. User logs in via website → receives session token
2. App stores session token securely (Capacitor `@capacitor/preferences`)
3. On app open (if logged in): `GET /api/sync` → merge into local DB (server wins if `updated_at` is newer)
4. On any local write (book import, position update, highlight, settings change): debounced `POST /api/sync` with the updated rows

### Conflict resolution
- **Settings**: last-write-wins by `updated_at`
- **Position**: last-write-wins (most recent read position wins)
- **Books**: merge by `book_id`; if same ID exists on both sides, take the one with newer `updated_at`
- **Highlights**: merge by `highlight_id`; deletions need a tombstone flag or soft-delete
