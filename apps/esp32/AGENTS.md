# ESP32 RSVP Reader

MicroPython firmware for ESP32 + ST7789 display. Shows books word-by-word with focal letter highlighting, acceleration, and WiFi/BLE management.

For project overview, roadmap, and shared settings see `../../agents.md`.

## Architecture

Single-loop state machine with two states: **idle** | **reading**. BLE transfer progress is drawn as an overlay on any state. The `App` class in `src/app/__init__.py` owns the loop; it polls BLE, then dispatches to the current state's tick handler. `RSVPReader` is a thin helper — no loop, no pause logic, no button handling. The state machine calls `reader.display_next_word()` and `reader.get_word_delay()` each tick. In idle, auto-shutdown turns the display off after a timeout and blocks until the button is pressed.

## Hardware

- ESP32 + ST7789 display — physically 170x320, driver 240x320, `rotation=3`
- BOOT button on GPIO 0 (only physical control)
- 3.7V LiPo via JST 1.25mm, onboard charging (TP4056), charges via USB
- No voltage divider on board — battery monitoring requires hardware mod (see Battery section)

## Key Classes

**App** (`src/app/__init__.py`)
- Single-loop state machine: idle → reading → paused
- `run()` — infinite loop: polls BLE, dispatches to `_tick_idle`, `_tick_reading`, `_tick_paused`
- Handles BLE transfer overlay and auto-shutdown on idle
- Short press: start / pause / resume

**Homescreen** (`src/app/homescreen.py`)
- Pure drawing functions, no state
- `draw_homescreen(display, title, progress_pct, wpm, paused)` — title, progress bar, stats, hint
- `draw_transfer_progress(display, pct, bytes_done, total_bytes)` — full-screen BLE transfer overlay
- Clean minimal design on 320x170 display (16x32 font)

**DisplayManager** (`src/hw/display.py`)
- Core display primitives: `clear()`, `fill_rect()`, `clear_rect()`, `text()`, `centered_text()`, `hline()`, `vline()`
- RSVP word rendering: `show_word(word)` → draws with focal letter highlighting, returns (x, width)
- `clear_word(x, width)` — clears previous word area
- `show_centered_message(msg)` — convenience full-screen message
- `shutdown()` / `wakeup()` — backlight control

**ButtonHandler** (`src/hw/button.py`)
- Debounced short/long press detection on GPIO 0
- `check_press_state(long_ms)` — returns `'short'`, `'long'`, or `None`
- `wait_for_press()` — blocking wait

**RSVPReader** (`src/reader/rsvp.py`)
- Thin helper — no loop, no pause, no button handling
- `load_text(text=None, resume=True)` — opens book or sample text
- `display_next_word()` — draws next word, returns word str or None at end
- `get_word_delay(word)` — delay in seconds including acceleration ramp
- `reset_acceleration()` — called on resume from pause
- `save_position()` / `get_progress_pct()` / `cleanup()`
- `gc.collect()` every 100 words to prevent fragmentation

**WordReader** (`src/reader/storage.py`)
- Streams words one at a time — never loads full file into RAM
- `open(byte_position)` — seeks instantly to saved position
- `next_word()` — returns next word with buffering
- `get_position()` — current byte offset for saving
- Buffer capped at 1000 chars with error recovery

**TextStorage** (`src/reader/storage.py`)
- Saves/loads the single book (`book.txt`) and byte position (`position.txt`)
- `load_position(word_offset)` — optionally rewinds N words on resume
- `.bak` backup system prevents position corruption on write failure

**BLE Server** (`src/ble/`)
- `server.py` — thin GATT dispatcher; registers 4 characteristics, delegates all I/O to handlers
- `handler_settings.py` — Settings characteristic (R/W): read config JSON, write updates `config_override.py` + soft reset
- `handler_position.py` — Position characteristic (R/W): read/write `position.txt`
- `handler_file_transfer.py` — File Transfer characteristic (Write + Notify): START/CHUNK/END state machine, base64 chunks, CRC32 verify, `book.tmp` → `book.txt`. Exposes `get_progress()` → `(in_progress, bytes_done, total_bytes)` for transfer overlay
- `handler_storage.py` — Storage characteristic (R): reads `os.statvfs('/')`, returns `{"free_bytes": n, "total_bytes": n}`
- `ble_config.py` — generated from `packages/ble-config`; do not edit manually
- Stops advertising when BLE resource is needed elsewhere
- `server.check_settings_updated()` / `server.check_transfer_completed()` / `server.get_transfer_progress()` — polled by App

## Dev Mode

Create `devmode` file on device. Boot shows "DEV MODE" and skips auto-start — safe for uploading code without the reader interfering.

## Build & Deploy

**Toolchain:** `mpy-cross` compiles `.py` → `.mpy` (MicroPython bytecode). Faster imports, smaller RAM footprint. `mpremote` handles serial communication and file transfer. Both installed via `pip install -r requirements.txt` (setup.sh does this automatically into a `.venv`).

**Upload flow** (`scripts/upload.sh`):
1. Compiles all `src/**/*.py` → `dist/src/**/*.mpy` via `mpy-cross -march=xtensa`
2. Cleans `src/` on device (recursive delete — prevents stale `.py`/`.mpy` conflicts)
3. Uploads `boot.py`, `main.py` as plain `.py` (entry points)
4. Uploads `dist/src/` tree via `mpremote fs cp -r` (compiled `.mpy`)
5. Does NOT reset the device — use `run.sh` to start the app

**Important:** `upload.sh` must always clean the device's `src/` before uploading. If both `.py` and `.mpy` exist for the same module, MicroPython prefers `.mpy` — stale `.mpy` files from a previous upload will shadow newer `.py` files, causing hard-to-debug issues (serial connection drops, silent crashes).

**`mpy-cross` version must match the firmware version.** Firmware is MicroPython v1.27.0. The `requirements.txt` pins `mpy-cross` which installs v1.27.0. If the firmware is ever updated, update `mpy-cross` to match.

**Dev workflow:**
1. Edit source files under `src/`
2. `bash scripts/upload.sh` — compiles + uploads (device must be in dev mode / REPL)
3. `bash scripts/run.sh` — starts the app via `exec` with `force_run=True`, stays blocking so you see serial output
4. Ctrl+C to stop, edit, repeat from step 2

## MicroPython Gotchas

- `from src import module` does **not** work — must add `src` to `sys.path`
- Both `boot.py` and `main.py` run automatically on startup
- Binary files: write with `'wb'`, read with `'rb'` then decode (handles non-UTF8)
- Socket cleanup is fragile — always `close()` + `gc.collect()` + 0.2s delay between requests
- Use `round()` not `//` for position percentage (avoids off-by-one)
- Never load large files into memory — always use `WordReader` streaming
- Device must be connected **directly** to the PC via USB — USB hubs/switch boxes can cause `OSError: [Errno 5] Input/output error` during serial communication

## Memory (~512KB RAM)

| Component | Usage |
|-----------|-------|
| BLE stack | ~30–40KB |
| WordReader buffer | ~1KB |
| JSON payload | ~100 bytes |
| Everything else | remaining, managed with gc.collect() |

Call `gc.collect()` before/after BLE ops and every 100 words during reading.

## Battery & Power

**Current state:** no voltage monitoring (GPIO 34/35/36/39 all read 0V — no voltage divider on board). Blue LED indicates charging when USB connected.

**To add battery level display:** solder 2x 100kΩ resistors as voltage divider: `Battery+ → [100kΩ] → GPIO35 → [100kΩ] → GND`. Halves 3.0–4.2V to 1.5–2.1V (safe for ESP32 ADC).

**Power consumption (1000mAh battery):**

| Mode | Draw | Battery life |
|------|------|-------------|
| Active reading | ~80–120mA | ~8–10h |
| Idle (display off) | ~20–40mA | ~25–50h |
| Deep sleep | ~10–20μA | months |

Deep sleep is implemented. Two-stage idle: display turns off after `AUTO_SHUTDOWN_TIMEOUT` (30 s default); if the button is not pressed within `DEEP_SLEEP_TIMEOUT` (5 min default), `machine.deepsleep()` is called with GPIO 0 as wake source. BLE is cleanly deinitialized before sleep (`ble.deinit()`). On wake, boot proceeds normally through `main.py` — position is preserved in `position.txt` across deep sleep.

## Common Issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| Shows sample text after upload | Binary read failed | Use `'rb'` mode + decode |
| Position not resuming | Byte offset not saved | Check `position.txt` contents |
| WiFi page times out | HTML sent in one block | Stream template line-by-line |
| Memory error on large books | File loaded into RAM | Use `WordReader` streaming |
| Upload fails silently | Socket not cleaned up | `gc.collect()` + 0.2s delay |
| Position file deleted | OOM during save | `gc.collect()` before file write |
